## Takeaway

Sequential read is the clear Orca bottleneck:

- Docker cold sequential is about 4x faster than Orca: 227 MiB/s vs 52-56 MiB/s.
- Docker warm sequential is page-cache dominated and much faster than Orca.

Random reads are more nuanced:

- With host cache dropped, Docker random is worse than Orca on this VM: 618 IOPS vs 1277-1439 IOPS.
- With warm host cache, Docker random is orders of magnitude faster because it is served from the host page cache.

The fair interpretation is that Orca's current block path is hurting sequential throughput badly. Docker also benefits from the host page cache in normal warm runs, which Orca does not currently match across the Firecracker/NBD/userspace-storage boundary.
